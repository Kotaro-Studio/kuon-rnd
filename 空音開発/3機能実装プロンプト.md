# 3機能を任意の Next.js プロジェクトに実装するプロンプト

> このプロンプトを Claude Code や Claude に渡すだけで、空音開発（kuon-rnd.com）で稼働中の
> 3つの機能を別のプロジェクトにも実装できます。
> 技術スタック前提：Next.js App Router + TypeScript + Cloudflare Workers（Hono）+ KV

---

## プロンプト本文（ここから全文コピーして渡してください）

---

以下の3つの機能を、このプロジェクトに実装してください。
すべて本番実績のある設計パターンで、空音開発（kuon-rnd.com）で稼働中のものと同じアーキテクチャです。

技術スタック前提：
- フロントエンド：Next.js App Router（TypeScript）
- バックエンド：Cloudflare Workers（Hono フレームワーク）+ KV
- 認証：JWT（HS256）を HttpOnly Cookie に格納
- メール送信：Resend API（マジックリンク認証）
- スタイリング：CSS-in-JS（インラインスタイル）

各機能は独立しているので、必要なものだけ実装しても構いません。

### 機能①：JWT 自動リフレッシュ（Notion 方式・永久ログイン）

#### 目的
ユーザーが月1回でもサイトを訪れる限り、永久にログイン状態が維持される。Notion と同じ UX。

#### アーキテクチャ

```
ページ読み込み
  │
  ▼
providers.tsx の TokenRefresher コンポーネント（1回/セッション）
  │
  ▼
POST /api/auth/refresh（Next.js API Route プロキシ）
  │ ※ HttpOnly Cookie から kuon_token を読み取り、Worker に Bearer で転送
  ▼
Auth Worker POST /api/auth/refresh
  │ ① 現在の JWT を検証（有効ならOK、期限切れなら 401）
  │ ② ユーザーの最新データを KV から取得（プラン変更の反映）
  │ ③ 新しい 30日 JWT を発行
  ▼
API プロキシが Set-Cookie で新しい JWT を HttpOnly Cookie にセット
  │
  ▼
フロントの localStorage も最新ユーザー情報に更新
```

#### Worker 側の実装（Hono）

```typescript
// POST /api/auth/refresh — サイレントトークンリフレッシュ
app.post('/api/auth/refresh', async (c) => {
  const auth = c.req.header('Authorization');
  if (!auth?.startsWith('Bearer ')) return c.json({ error: 'No token' }, 401);

  const payload = await verifyJWT(auth.slice(7), c.env.AUTH_SECRET);
  if (!payload) return c.json({ error: 'Invalid or expired token' }, 401);

  const raw = await c.env.USERS.get(`user:${payload.email}`);
  if (!raw) return c.json({ error: 'User not found' }, 404);

  const user = JSON.parse(raw);
  const now = Math.floor(Date.now() / 1000);
  const jwt = await createJWT(
    { email: user.email, plan: user.plan, iat: now, exp: now + 30 * 24 * 3600 },
    c.env.AUTH_SECRET
  );

  return c.json({ ok: true, jwt, user: { email: user.email, name: user.name, plan: user.plan } });
});
```

#### API プロキシ（Next.js catch-all route）

`refresh` パスの場合も、`verify` や `google` と同様に Set-Cookie ヘッダを付ける：

```typescript
// /api/auth/[...path]/route.ts 内
if ((pathStr === 'verify' || pathStr === 'google' || pathStr === 'refresh')
    && method === 'POST' && res.ok && data.jwt) {
  return new Response(JSON.stringify(data), {
    status: 200,
    headers: {
      'Content-Type': 'application/json',
      'Set-Cookie': `kuon_token=${data.jwt}; Path=/; HttpOnly; Secure; SameSite=Lax; Max-Age=${30 * 24 * 3600}`,
    },
  });
}
```

#### フロントエンド（providers.tsx に追加）

```typescript
function TokenRefresher() {
  const refreshedRef = useRef(false);

  useEffect(() => {
    if (refreshedRef.current) return;
    if (typeof window === 'undefined') return;

    const stored = localStorage.getItem('kuon_user');
    if (!stored) return; // ログインしていない

    refreshedRef.current = true; // セッション中1回だけ

    (async () => {
      try {
        const res = await fetch('/api/auth/refresh', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
        });
        if (!res.ok) {
          if (res.status === 401) localStorage.removeItem('kuon_user'); // 期限切れ→クリア
          return;
        }
        const data = await res.json();
        if (data.user) localStorage.setItem('kuon_user', JSON.stringify(data.user));
      } catch { /* ネットワークエラーは無視、次回訪問時にリトライ */ }
    })();
  }, []);

  return null;
}

// ClientProviders 内に配置
export function ClientProviders({ children }) {
  return (
    <LangProvider>
      <PageviewTracker />
      <TokenRefresher />
      {children}
    </LangProvider>
  );
}
```

#### 設計のポイント

- JWT 自体は 30日有効のまま変えない（セキュリティバランス）
- フロントが毎回ページ読み込み時に `/api/auth/refresh` を叩く（1セッション1回）
- Cookie が有効なら新しい 30日 JWT に更新される → 実質永久ログイン
- Cookie が期限切れなら 401 → localStorage をクリア → 自然にログアウト
- 管理者の作業ゼロ。すべて自動

### 機能②：多言語切り替えシステム（ブラウザ自動検出 + 手動切替）

#### 目的
初回アクセス時にブラウザ言語を自動検出し、ユーザーがいつでも手動で切り替えられる。
サイト全体で一貫した言語が適用される。

#### アーキテクチャ

```
初回アクセス
  │
  ├── localStorage に保存言語がある？ → その言語を使用
  │
  └── ない → navigator.language を解析して自動選択
       例：'ja' → 日本語、'ko' → 韓国語、'pt-BR' → ポルトガル語

ユーザーが言語を手動変更
  │
  └── localStorage に保存 → 次回以降は自動検出をスキップ
```

#### context/LangContext.tsx

```typescript
'use client';
import React, { createContext, useContext, useState, useEffect } from 'react';

// 対応言語を定義（プロジェクトに合わせて変更）
export type Lang = 'ja' | 'en' | 'ko' | 'pt' | 'es';

interface LangContextType {
  lang: Lang;
  setLang: (l: Lang) => void;
}

const LangContext = createContext<LangContextType>({ lang: 'ja', setLang: () => {} });

const VALID_LANGS: Lang[] = ['ja', 'en', 'ko', 'pt', 'es'];
const STORAGE_KEY = 'site-lang'; // プロジェクトごとに変更

function detectLang(browserLang: string): Lang {
  const l = browserLang.toLowerCase();
  if (l.startsWith('ja')) return 'ja';
  if (l.startsWith('ko')) return 'ko';
  if (l.startsWith('pt')) return 'pt';
  if (l.startsWith('es')) return 'es';
  return 'en'; // フォールバック
}

function getInitialLang(): Lang {
  if (typeof window === 'undefined') return 'ja'; // SSR 時のデフォルト
  const saved = localStorage.getItem(STORAGE_KEY) as Lang | null;
  if (saved && VALID_LANGS.includes(saved)) return saved;
  return detectLang(navigator.language || '');
}

export function LangProvider({ children }: { children: React.ReactNode }) {
  const [lang, setLangState] = useState<Lang>('ja');

  useEffect(() => { setLangState(getInitialLang()); }, []);

  const setLang = (l: Lang) => {
    if (typeof window !== 'undefined') localStorage.setItem(STORAGE_KEY, l);
    setLangState(l);
  };

  return (
    <LangContext.Provider value={{ lang, setLang }}>
      {children}
    </LangContext.Provider>
  );
}

export function useLang() { return useContext(LangContext); }

// 多言語テキストヘルパー型
export type L5 = Record<Lang, string>;
```

#### 言語切り替えUI（LangSwitcher コンポーネント）

ドロップダウン式。国旗 + 母語名 + 英語名を表示。2つのバリアント：

- `inline`（デフォルト）：ヘッダーやフッターに配置するドロップダウン
- `stacked`：モバイルメニュー用のピル型ボタン列

```typescript
const LANGS = [

  { code: 'ko', flag: '🇰🇷', name: 'Korean', nativeName: '한국어' },
  { code: 'pt', flag: '🇧🇷', name: 'Portuguese', nativeName: 'Português' },
  { code: 'es', flag: '🇪🇸', name: 'Spanish', nativeName: 'Español' },

```

機能：
- 外側クリックで閉じる
- Escape キーで閉じる
- アニメーション付き開閉（transform + opacity）
- 現在の言語にチェックマーク表示
- aria-label / aria-expanded / role="listbox" でアクセシビリティ対応

#### 各ページでの使い方

```typescript
import { useLang } from '@/context/LangContext';
import type { Lang } from '@/context/LangContext';

type L5 = Record<Lang, string>;
const t5 = (m: L5, lang: Lang) => m[lang] ?? m.en;

export default function SomePage() {
  const { lang } = useLang();

  return (
    <h1>{t5({
      ja: '日本語テキスト',
      en: 'English Text',
      ko: '한국어 텍스트',
      pt: 'Texto em Português',
      es: 'Texto en Español',
    }, lang)}</h1>
  );
}
```

#### 設計のポイント

- SSR 時は `'ja'` をデフォルトとし、`useEffect` でクライアント側の言語を適用（ハイドレーションエラー回避）
- `useLang()` は必ず `const { lang } = useLang()` で分割代入すること（直接 `const lang = useLang()` は型エラー）
- 言語追加は `Lang` 型、`VALID_LANGS`、`detectLang`、`LANGS` 配列の4箇所を変更するだけ
- フォールバック：`ko` や `pt` の翻訳が未定義の場合は `en` にフォールバック

### 機能③：オーナー管理ダッシュボード

#### 目的
サイトオーナーだけがアクセスできる管理画面。ユーザー管理 + ページビュー解析。

#### アーキテクチャ

```
/admin にアクセス
  │
  ▼
GET /api/auth/me で JWT 検証
  │ ① メールアドレスが OWNER_EMAIL と一致するか確認
  │ ② 不一致 → / にリダイレクト
  │ ③ 一致 → ダッシュボード表示
  │
  ├── GET /api/auth/admin/users → ユーザー一覧（ページネーション・検索・プラン絞り込み）
  ├── PUT /api/auth/admin/plan → ユーザーのプラン変更
  └── GET /api/auth/admin/pageviews?days=30 → ページビューデータ
```

#### Worker 側エンドポイント

app.post('/api/auth/pageview', async (c) => {
  // リクエストから path と country（CF-IPCountry ヘッダ）を取得

  const key = `pv:${today}`;

  // 既存データを取得 → カウントアップ → KV に保存（TTL 90日）
  const existing = await c.env.SESSIONS.get(key);
  const pv = existing ? JSON.parse(existing) : { total: 0, countries: {}, pages: {} };
  pv.total++;
  pv.countries[country] = (pv.countries[country] || 0) + 1;
  pv.pages[path] = (pv.pages[path] || 0) + 1;

function PageviewTracker() {
  const pathname = usePathname();

#### ダッシュボード UI 機能一覧

1. **時間帯別挨拶**（深夜/朝/昼/夜）+ 日替わりモチベーション引用文
2. **概要カード**（4枚）
   - 今日の PV
   - 7日間 PV
   - 30日間 PV
   - 週間成長率（前週比 %）
3. **PV 棒グラフ**（30日間・純 CSS、ライブラリ不要）
4. **マイルストーン進捗バー**（次の目標ユーザー数 / PV 数）
5. **国別アクセス**（国旗絵文字 + 国名 + 件数）
6. **人気ページ TOP 10**（パス + PV 数）
7. **ユーザー一覧テーブル**
   - 検索（名前 / メール）
   - プラン絞り込み（free / student / pro）
   - ページネーション（50件/ページ）
   - プラン変更ボタン（ドロップダウン）
   - 登録日・最終ログイン日表示
   - 国旗表示
8. **アクティブ日数ストリーク**（連続でPVがある日数）

#### セキュリティ

- JWT 検証 + メールアドレスが `OWNER_EMAIL` 定数と一致する場合のみアクセス許可
- 一般ユーザーが `/admin` にアクセスしても 403 でリダイレクト
- プラン変更は `confirm()` ダイアログで誤操作防止

#### 設計のポイント

- PV グラフは CSS だけで実装（棒の高さを最大値に対する割合で計算）。chart.js 等は不要
- 国旗は ISO コードから `String.fromCodePoint` で生成（画像不要）
- KV の `list()` でユーザー一覧を取得するため、大量ユーザー時はカーソルベースのページネーションが必要
- `OWNER_EMAIL` はハードコードでOK（環境変数にしてもよい）
- ダッシュボードのデザインはダーク基調（#0f172a 背景）が見やすい

### 共通の実装手順

1. Auth Worker に上記エンドポイントを追加
2. Next.js の catch-all API プロキシ（`/api/auth/[...path]/route.ts`）で全パスを Worker に転送
3. `context/LangContext.tsx` を作成し `app/providers.tsx` で `<LangProvider>` で囲む
4. `app/providers.tsx` に `<PageviewTracker />` と `<TokenRefresher />` を配置
5. `app/lang-switcher.tsx` を作成しヘッダー/フッターに配置
6. `app/admin/page.tsx` にダッシュボード UI を実装
7. Worker をデプロイ（`npx wrangler deploy`）
8. `git push` で Cloudflare Pages にデプロイ

### 環境変数

| 変数名 | 登録先 | 用途 |
|--------|--------|------|
| `AUTH_SECRET` | Worker | JWT の署名・検証に使用する秘密鍵 |
| `RESEND_API_KEY` | Worker or Pages | マジックリンクメール送信（認証機能がある場合） |

### KV Namespace

| Namespace | 用途 |
|-----------|------|
| `USERS` | ユーザーデータ（`user:{email}` キー） |
| `SESSIONS` | セッション、ページビュー、マジックリンクトークン等の一時データ |
