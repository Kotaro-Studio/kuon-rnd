## 🎨 視覚デザインの好み

### タイポグラフィ
- **明朝体 (Hiragino Mincho / Shippori Mincho)** をヒーロー・大見出しに
- **Helvetica Neue / Arial** をボディ・UI に
- 漢字の品格を活かす日本的アイデンティティ
- 大胆なフォントサイズ (clamp で 2.5rem〜5rem)
- 詰めすぎず・余白を大胆に

### カラー
- 基調色: **白基調** + **深い藍 (#0a1226)** ダーク基調
- アクセント: **ローズゴールド (#e8b8a0)** または **#0284c7 ブルー**
- 宝石パレット (Sapphire/Emerald/Amethyst/Gold) のような **品の良い差別化**
- グラデーション (linear / radial) を効果的に
- 派手すぎる色は避ける (Ableton Live 系の極彩色は NG)

### レイアウト
- **Apple 級の余白 (whitespace)** を尊重
- **モバイルファースト** + PC でも美しい
- max-width 1200-1400px で読みやすさ確保
- カードベースの情報整理 (グリッドレイアウト)
- 1 セクション 1 メッセージの原則
- `clamp()` を多用してレスポンシブ徹底

### アニメーション
- **hover 効果** (translateY -2px〜-4px + boxShadow)
- 滑らかな 0.2-0.4s ease 遷移
- 過剰なアニメーションは避ける
- ガラスモーフィズム (backdrop-filter blur) を控えめに

### 装飾
- **Bravura SMuFL フォント**で楽譜記号 (𝄞 等) を薄く配置
- グラデーションオーブ (radial-gradient で背景装飾)
- 「日本の侘び寂び」感
- 過剰な絵文字は避けるが要所で使う

## 📝 コピーライティングの好み

### ヒーローコピー
- **明朝体での詩的な短いフレーズ** が刺さる
- 例: 「音楽家のための、ひとつの場所。」「あなたのブラウザに、スタジオが届く日。」
- アイデンティティ即時活性化 (「音楽家の」「あなたの」)
- 「ひとつ」「すべて」「次の」などのパワーワード
- 句読点で韻律を作る (例: 「練習も、録音も、配信も。」)

### サブコピー
- **具体的な数字を必ず入れる** (30+, ¥780, $300/年, 4 年間)
- 競合比較を盛り込む (Pro Tools 代替, Logic Pro 代替)
- ブラウザ完結 / インストール不要 / 完全無料を強調

### CTA
- 「→」矢印で前進感
- 摩擦撤去文言 (30 秒 / メールだけ / ワンクリック解除)
- 動詞中心の即決促す表現

### 多言語対応
- 必ず ja / en / es / ko / pt / de の **6 言語対応**
- L5 型 (Partial<Record<Lang, string>> & { en: string }) を使用

## 🧠 IQ190 心理学的フックの好み

オーナーは以下のフックを「最強」と認識:

1. **Identity Activation** - 「音楽家の」「あなたの」で即時自己同一化
2. **Specificity Bias** - 抽象でなく具体数字 (¥780/月、4 年間、30+ 等)
3. **Loss Aversion** - 「未登録だと使えるアプリが限られる」
4. **Reciprocity** - 透明な価値開示 (「6 つできること」を隠さない)
5. **Friction Removal** - 「メールだけ・30 秒・ワンクリック解除」
6. **Social Proof** - 数字で信頼 (35+ ツール、8 カテゴリ等)
7. **Authority** - 朝比奈幸太郎の権威性 (P-86S 設計者・元音大生)
8. **Curiosity Gap** - 「30 のツールを覗く」式の表現
9. **Privacy Trigger** - 「クラウド送信ゼロ」の強い訴求
10. **Sunk Cost** - 「練習履歴・成長グラフ」「累計 N 時間」
11. **Daily Habit Loop** - 「今日の名言」「毎日の儀式」
12. **Brand Coherence** - 楽譜記号・宝石色等の継続的シンボル
13. **Status Signaling** - メンバー番号 No. 1,247
14. **Streak (Duolingo)** - 連続日数の可視化

## 📐 ペルソナ設計の好み

ペルソナは **具体性 > 数** が原則:

### 必ず含めるペルソナ
1. **演奏家** (Musicians)
2. **音大生** (Music Students)
3. **歌い手・配信者** (Singers & Streamers / 歌ってみたが多い層)
4. **録音エンジニア** (Recording Engineers / Kotaro 本人と同類)
5. **音楽ファン** (Music Fans)

### 各ペルソナの記述パターン
- **問題認識** (このペインに覚えがある?)
- **具体的な数字でベネフィット**
- **競合との比較**
- **アイデンティティ確認** (「これ、自分のことだ」)

例:
> 演奏家:「練習録音のコンピング、LUFS マスタリング、AI 音源分離。Pro Tools $300/年・Logic ¥30,000 の機能が、ブラウザで完全無料。」

## 🎯 製品哲学の好み

### コア哲学
- **「ワンポイント録音哲学」**: 録ったままに、美しく整える
- **タイムストレッチ・ピッチ補正等の作為的処理は意図的に省略**
- **安定性 > 機能数**
- **シンプル UI > 多機能 UI**
- **完全無料・登録のみ** が基本
- **クラウド送信ゼロ・プライバシー第一**
- **ブラウザ完結 = インストール不要**

### 技術選定
- **Web Audio API** + Canvas + IndexedDB の組み合わせ
- WebAssembly で重い DSP (Rust ベース)
- **lamejs** で MP3 エンコード (既存)
- 適切に Edge Runtime / Server Component を使い分け
- Cloudflare Pages + Workers + R2 が前提
- **「サーバーコストゼロ」を可能な限り優先**

### サブスク設計思想
- Free プラン充実が集客の核
- Prelude/Concerto/Symphony/Opus 4 段階
- HALF50 初月割引が標準
- 年払い 2 ヶ月無料が標準
- Curanz Sounds との連携は **保留中**
  (`空音開発/labelization-discussion.md` 参照)

## 🌐 SEO + GEO の好み

各 LP に必ず:
- **80+ ユニークキーワード** (4 軸 × 3 言語以上)
  - 高ボリューム汎用語
  - 中ボリューム用途別
  - ロングテール音楽特化
  - 競合比較
- **3 種の JSON-LD**:
  - SoftwareApplication
  - HowTo (具体的なユースケース手順)
  - FAQPage (8 Q&A 程度・AI 引用最適化)
- ChatGPT/Perplexity/Gemini が引用しやすい構造
- 競合名 (LALAL.AI / Calm / Pro Tools / Audacity 等) と直接比較
- 朝比奈幸太郎の権威性を必ず明記

## 🚫 やってはいけないこと

- ❌ **音大不要論を語る** (CLAUDE.md §37 沈黙の勝利戦略)
- ❌ Curanz Sounds と Kuon を混同して扱う
- ❌ Curanz Collective レーベル化を勝手に進める (議論保留中)
- ❌ 過剰な絵文字や派手な装飾
- ❌ 既存のアプリページの音声処理ロジックに触る (CLAUDE.md §19)
- ❌ "完全無料" を否定する仕様 (Free プランの基本ツールは無制限)
- ❌ プラン名を変えない (Prelude/Concerto/Symphony/Opus は確定)
- ❌ サブレーベル名を勝手に決める

## ✅ 必ず確認すること

- 全 LP は SEO + GEO 標準準拠
- 全コードは npx tsc --noEmit で型チェック通過
- 全コミットメッセージは英語 + 日本語混在で詳細に
- ファイル変更後は必ずデプロイコマンドを提供
- 大規模変更後は事後報告 (オーナーが見る前提)
- ブラウザアプリは AuthGate ルールに従う
  - Free no-login allowlist は `pricing-display.ts` で管理
